## Planificación

Aunque no se utilizó Stitch, se planificó la historia antes de escribir código:

- La historia se dividió en secciones: Beginning, Middle, End.
- Los commits se planificaron de la siguiente manera:
  - `docs: create basic HTML structure` → crear la estructura HTML de la historia.
  - `docs: add author` → añadir el nombre del autor al final.
  - `style: apply CSS` → aplicar estilos y diseño a la historia.
  - `docs: add final screenshot` → añadir captura de pantalla al README.
- Esta planificación ayudó a mantener los commits atómicos y organizados.
